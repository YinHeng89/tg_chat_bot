Everything You Need to Build Smart Telegram Bots

Multi-Bot Hot Management

Get Started

Deploy in Seconds

Choose the deployment method that fits your workflow.

Docker (Recommended)

Shell 
# Clone the repository
git clone https://github.com/yinheng1998/tg_chat_bot.git
cd tg_chat_bot

# Start the service
docker compose -f docker-compose.yml up -d

# Access web panel at http://localhost:8000

Production Deployment

Shell 
# Use pre-built image with health checks
docker compose -f docker-compose.prod.yml up -d

# The image supports both amd64 and arm64

Local Development

Shell 
# Backend
pip install -r requirements.txt
python main.py

# Frontend (separate terminal)
cd frontend
npm install
npm run build
After deployment, visit http://localhost:8000 to access the web admin panel. Set your admin password on first login, then add your Telegram Bot Token to get started.

Frequently Asked Questions

The system supports 12+ providers: OpenAI, DeepSeek, Anthropic Claude, Google Gemini, Alibaba Qwen (Tongyi Qianwen), Moonshot, Zhipu AI (GLM), StepFun, MiniMax, Baidu Wenxin (ERNIE), ByteDance Doubao, and Ollama for local models. Each Bot can be configured with a primary and a backup model with automatic fallback.

Yes. The system supports unlimited Bot instances running simultaneously. Each Bot has its own personality, model configuration, group chat settings, and plugin preferences. You can add, remove, start, or stop bots at any time through the web panel without restarting the service.

Each plugin exposes a tool definition in OpenAI Function Calling format. When a user sends a message, the tool definitions are sent alongside the conversation. The LLM decides whether to call a tool. The system executes the tool and feeds the result back to the LLM for a final response. Users can also trigger plugins manually with commands like /search, /weather, /calc, /translate, and /draw.

Conversation history is stored in SQLite with isolation per bot_id + chat_id. When a conversation exceeds 30 messages, the system automatically triggers an LLM-based summarization, compressing older messages while retaining the last 8 messages and a summary of prior context. This balances context preservation with token efficiency.

The system implements multiple layers of security: JWT-based authentication with 24-hour token expiry, bcrypt password hashing, a 12-character recovery code for password reset, blacklist for blocking users, per-user rate limiting, whitelist mode option, CLI sandbox with 256MB memory limit and dangerous-command blacklist, and automatic log redaction of API keys and tokens.

Creating a new plugin is straightforward: inherit from the PluginBase class, implement the execute() method and get_tool_definition() method, then register it in the plugin registry. The system handles the rest — your plugin will automatically appear in the Function Calling tool list and can be enabled/disabled from the web panel.
