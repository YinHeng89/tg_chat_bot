Security Design

Multiple layers of protection for production deployment

Password & Recovery

bcrypt hashed passwords with auto-migration from plaintext. Recovery code (12-char) for password reset. 6-char minimum length.

JWT Authentication

HS256 tokens with 24h expiry. Env-injected or auto-generated secret key. All /api/* routes require Bearer token.

CLI Sandbox

Dangerous command blacklist (shutdown, mkfs, fork bomb). Resource limits: 256MB RAM, 30s CPU timeout.

Log Redaction

loguru filter auto-masks API keys, JWT tokens, and passwords in all log output. 7-day retention with zip compression.

Deploy in 3 Steps

Single Docker container, ready in minutes

1. Create .env file

WEB_JWT_SECRET=$(openssl rand -hex 32) # Optional: MEMOS_API_URL=... # Optional: MEMOS_API_KEY=...

2. docker compose up -d

docker compose -f docker-compose.prod.yml up -d

3. Open Web Panel & Setup

http://your-server-ip:8025 -> Set admin password -> Save recovery code -> Add LLM model -> Add Bot token -> Start chatting

Docker

Multi-stage build (Node + Python). python:3.11-slim base. Exposed port 8000. Healthcheck via /healthz.

Volumes

Three persistent volumes: data/ (SQLite DB), logs/ (rotated logs), workspace/ (CLI sandbox). Data survives container updates.

Multi-Arch

Pre-built images for linux/amd64 and linux/arm64. Published to Docker Hub. Build with docker buildx.
